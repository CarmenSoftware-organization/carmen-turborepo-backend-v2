================================================================================
  CRITICAL BUG FOUND: Gateway returns HTTP 201 for failed microservice calls
================================================================================

  Many application POST endpoints in the gateway return HTTP 201 even when the
  microservice operation fails. The response body correctly shows the error:
    { "data": null, "status": 500, "success": false, "message": "..." }

  But the HTTP status code is 201 (Created), which misleads API consumers.

  Root Cause:
    The gateway controller creates the response BEFORE checking the microservice
    result. The controller should check response.response.status !== HttpStatus.CREATED
    before returning 201.

  Affected endpoints (confirmed):
    - POST /api/:bu/price-list
    - POST /api/:bu/spot-check
    - POST /api/:bu/transfer
    - POST /api/:bu/good-received-note
    - POST /api/:bu/stock-in
    - POST /api/:bu/stock-out
    - POST /api/:bu/physical-count
    - POST /api/:bu/physical-count-period
    - POST /api/:bu/period
    - POST /api/:bu/request-for-pricing
    - POST /api/:bu/purchase-order
    - POST /api/:bu/price-list-template
